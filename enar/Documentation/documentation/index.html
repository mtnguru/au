Overview

Enar is the first design-driven Multipurpose DRUPAL 9 Theme with great attention to details, incredible features, an intuitive user interface and everything else you need to create outstanding websites. Whether you are launching a creative agency site, a business website, a blog or portfolio.

Enar is the ultimate tool for creating immersive websites. Be amazed how quickly you get your next website up and running. It does not matter if you want to sell products or attract new readers, Enar offers limitless customization options, drag & drop layouts and headers with an easy to use Theme Setting. Enar is the perfect combination of a beautiful and professional design with tons of powerful features and a bulletproof code base.

 

Requirements For Enar

Database

MySQL 5.5.3/MariaDB 5.5.20/Percona Server 5.5.8 or higher with PDO and an InnoDB-compatible primary storage engine, PostgreSQL 9.1.2 or higher with PDO, SQLite 3.6.8 or higher.

PHP

PHP 5.5.9 or higher

Recommended PHP Configuration

Limits Many issues that you may run into such as; white screen, demo content fails when importing, empty page content with Fusion Builder and other similar issues are all related to low PHP configuration limits. The solution is to increase the PHP limits. You can do this on your own, or contact your web host and ask them to increase those limits to a minimum as follows:

max_execution_time 180

memory_limit 512M

Installation

Demo installation

The quickstart package came with theme helps you to build a copy of our live demo. It is the highly recommended way for newbies or if you don’t want to go through the individual setup of each module and theme.

STEPS to do:

  1. Create a new database in your phpmyadmin/cPanel tools and create database user.

  2. Import the supplied Demo.sql.zip or Demo-shop.sql.zip file (located in Database folder in your package) into this database.

  3. Upload the "Demo.zip" or "Demo-shop.zip" folder to your server. In most cases you will unpack it under the public_html folder, www root folder (windows hosting) or the root folder of your server.

  4. Change the database info in sites/default/settings.php to match the host, database name, user and password for your new created database. Done.

  5. You can login to manage site folow url: http://domain.com/user with the default admin account is: username: admin, password: admin.

 

Existing site installation

The option is used when you have a fresh or existing Drupal site. When you finish steps below, content types and view are avaible.

STEPS to do:

  1. Copy both the enar  and innovation themes (located in package at Packages/themes) into your sites themes directory.

  2. Copy all modules  module from Packages/modules to your sites modules.

  3. Copy libraries from Packages/libraries to libraries folder.

  4. Enable the Enar and set it to default theme from Appearance menu. http://Yoursiteurl.com/admin/appearance.

  5. Enar all modules in this modules folder with order below:

    1. Enable Feature module
    2. Enable all modules Innovation Theme group
    3. Enar module with prefix enar_

NOTE: Configure for shortcode is not ready when you install site with Existing site installation. You should read Enar Shortcodes section to do it manually.

Theme Settings

Enar is a sub theme extends from Innovation base theme using Bootstrap 3 which provides the ability to build any responsive layout you can think of by using simple actions such as Drag & Drop elements on UI. It also supports to create multiple layouts, so you can show your pages with different shapes very easily.

Go to http://YOURSITEURL.COM/admin/appearance/settings/enar to manage your theme settings.

1. Layout Settings

When you installing Enar theme, it has a layout called Default contains list of sections and regions to render content to page.

Default layout will be assigned to all pages. If you want some special pages have different Default layout, just create new layout and assign page to layout as step below:

Step1: Choose layout which you want to assign to page

Step 2: Scroll down to Pages Assignments textarea

Step3: Input path into this textarea (ex: /node/30)

Step 4: Don't forget save to apply your setting

Section settings

Section is a container to put regions of page. You can re-order sections or move regions in section to other sections.

Ex: If you want banner section to appear above header section, you can drag the banner section above the header and save the settings. Saving these settings will automatically apply these changes when you view the front end of your site. Additional, you can customised section through:

  • Full Width: this setting provides the ability to make sections into a full width responsive layout, so that the content will be the full width of the browser. It is usually use in cases show slider full width or region have full width background .

  • Visible: this setting provides visibility section based on the browser size

  • Background color: this setting provides the ability to use color code to set individual sections with a background colour which applied to them, without requiring any use of CSS.

  • Stick on top: this setting make the section stick to the top of the screen permanently per page

  • Custom class: this setting allows user apply owner class to style section

  • Custom column padding: this setting allows user set again padding of columns in section. Default, each columns in section padding 15px. You can set to 0 (no padding).

Region settings

Region is a container to put or group block drupal. With Innovation base theme using Twitter boostrap, you can set width for region also drag & drop it on layout.

 

2. Color Settings

The Preset settings provide the functionality to create your own colors so that you can have full control over the color scheme of your website using a hex code.
STEPS to set pernament color for the theme:

Step 1: Click on Presets vertical tab at the left side

Step 2: Choose color in dropdown labeled Default preset

Step 3: Don’t forget to save your changes to apply them.

 
In each of color you can define color for elements such as heading, link, text and use variable less as below in your less file. 
 
Base Color: (@ base_color)
 
Link Color: (@link_color)
 
Link hover Color: @link_hover_color
 
Text color: @text_color
 
Heading Color: @heading_color

3. Font Settings

The Font settings under Font vertical tab allow you setting font and size for all site and elements using Google Font

Blog

Enar theme is using Article content type to show blog posts. It offers many different options for you to present your blog posts, along with several different blog layouts.

1. Blog styles

You should read this article https://www.drupal.org/node/774798 to understand about view mode concept if you haven't understanded it yet.

Except 2 default view modes are Teaser & Default is avaiable for every content type. We have created 2 new view modes called Teaser Grid  and Teaser Grid 2 for Article content type. There are 4 blog styles in Enar theme

a. Standard Blog

Display blog items with Teaser view mode in blog view.

Template file called node--article--teaser.html.twig (themes/enar/templates/node)  is used to decorate 

b. List Blog

Display blog items with Teaser view mode in blog view.

This file also using template file called node--article--teaser.html.twig (themes/enar/templates/node) same Standard Blog. But this view block has a wrapper class blog-list in view.

c. Grid view Style 01

Display blog items with Teaser Grid view mode in blog view.

Template file called node--article--teaser_grid.html.twig (themes/enar/templates/node)  is used to decorate blog item in this view.

d. Grid view Style 02

Display blog items with Teaser Grid 2 view mode in blog view.

Template file called node--article--teaser_grid2.html.twig (themes/enar/templates/node)  is used to decorate blog item in this view.

2. Blog Singe Page

Blog single page used Default view mode to render page. To access to this view mode, go to http://YOURSITEURL/admin/structure/types/manage/article/display (*). At here, you can see fields and how to display it.

Enar theme is using template file called node--article.html.twig located at (themes/enar/templates/node) to decorate blog post in single page. So, you can modify this file to customize to suitable to your requirements.

To render a field in template file, the first you must make this field not Hidden in Manage display view mode (*). After that, using syntax: {{ content.FIELD_NAME}} to render.

Ex: to render image => {{ content.field_image }} or  render category {{ content.field_blog_categories }}

3. Blog image setting

To display carousel images, you must install Carousel Formatter module (enar/modules/carousel_formatte) before.

STEP 1: Go to http://YOURSITEURL/admin/structure/types/manage/article/display 

STEP 2: Choose your view mode (Ex: Default)

STEP3: Set Carousel Foratter in Image field

STEP 4: Configure carousel

Reference to Carousel /Formatter module at section Enar Preminum module for more detail.

4. Blog widgets

There are 4 block widgets for Blog. So, you just put it into region which you want.  

- Recent Post

- Category Post

- Popular Post

4. Related Portfolio

Related Projects are portfolio posts and appear below each active portfolio post, and they are posts that have similar categories as the one being currently viewed. Related Projects are a convenient way to show your viewers other projects that they may be interested in, based on the one they are currently viewing.

Block Portfolio: Related Porfolio is available in Portfolio View.

5. Portfolio Filter

Portfolio Filters are most commonly used to sort your Portfolio items. You can use categories taxonomy that you’ve assigned to portfolio as filters. 
 
These filters allow your viewers to view your works in an organized manner, and to sort through your portfolio content on a particular page based on the filter they’re interested in.
 
Show/hide portfolio filter:
 
STEP1: Edit your portfolio view 
STEP2: Click on Settings link of Innovation Grid View
STEP 3: Choose (Select) to hide filter or choose taxonomy you want to filter
